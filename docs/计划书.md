# 多平台电商自动化系统 — 计划书

## 一、项目目标

构建一套完整的**商品采集 → 数据存储 → 多平台上架**自动化系统：
- 从**飞书多维表格**读取商品 URL
- 根据 URL 自动识别来源平台（淘宝/天猫、3e3e 等），执行对应采集器
- 采集**主图、价格、商品参数**
- 将数据统一存储
- 自动上架到**微信视频号小店、小红书、抖店**，自动填写各平台所需字段

## 三、模块设计

### 3.1 数据源模块 (`datasource/`)

#### 3.1.1 飞书 API 方式
```
datasource/
├── feishu_api.py        # 飞书开放API对接
└── feishu_excel.py      # 导出Excel本地读取（备用方案）
```

**飞书多维表格字段设计：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 商品URL | URL | 采集源链接 |
| 来源平台 | 单选 | 自动识别或手动选：淘宝/天猫/3e3e |
| 目标平台 | 多选 | 微信视频号/小红书/抖店 |
| 加价规则 | 文本 | 如 "+30%" 或 "+15" 或留空后期配置 |
| 采集状态 | 单选 | 待采集/采集中/已采集/采集失败 |
| 上架状态 | 多选 | 视频号:已上架 / 小红书:待上架 / ... |
| 采集时间 | 日期 | 自动填写 |
| 商品标题 | 文本 | 采集回填 |
| 采集价格 | 数字 | 采集回填 |
| 上架价格 | 数字 | 加价后计算回填 |

**飞书 API 关键接口：**
- `GET /open-apis/bitable/v1/apps/:app_token/tables/:table_id/records` — 读取记录
- `PUT /open-apis/bitable/v1/apps/:app_token/tables/:table_id/records/:record_id` — 更新状态回写
- 认证方式：tenant_access_token（需在飞书开放平台创建应用）

#### 3.1.2 Excel 备用方式
- 从飞书导出 `.xlsx` 到桌面
- 使用现有 `utils/excel.py` 读取
- 采集完成后手动导入回飞书

**采集器基类设计：**

```python
class BaseCrawler:
    """所有采集器继承此类"""

    async def crawl(self, url: str) -> ProductData:
        """采集单个商品，返回统一数据结构"""
        raise NotImplementedError

    def match_url(self, url: str) -> bool:
        """判断此URL是否属于本采集器"""
        raise NotImplementedError
```

**URL 自动分发逻辑：**
```python
# registry.py
CRAWLER_REGISTRY = [
    (r"(taobao|tmall)\.com", TmallCrawler),
    (r"3e3e\.\w+", E3E3Crawler),      # 待确认网址
    # 后续新增采集器只需在此注册
]

def get_crawler(url: str) -> BaseCrawler:
    for pattern, crawler_cls in CRAWLER_REGISTRY:
        if re.search(pattern, url):
            return crawler_cls()
    raise ValueError(f"不支持的URL: {url}")
```

**统一数据结构：**
```python
@dataclass
class ProductData:

    price: float                  # 采集价格

    source_url: str               # 来源URL
    source_platform: str          # 来源平台
    crawl_time: str               # 采集时间
```

### 3.3 数据存储层 (`storage/`)

```
storage/
├── __init__.py
├── database.py          # SQLite 数据库操作
├── models.py            # 数据模型定义
└── image_store.py       # 图片文件管理
```

**为什么用 SQLite 而不是 Excel：**
- 多平台上架需要频繁查询和更新状态
- 支持按平台/状态/时间筛选
- 并发安全，不会出现 Excel 文件锁定问题
- 零部署成本（Python 内置）

**数据表设计：**

```sql
-- 商品主表
CREATE TABLE products (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    source_url  TEXT UNIQUE NOT NULL,
    platform    TEXT NOT NULL,           -- taobao/tmall/3e3e
    title       TEXT,
    price       REAL,
    params      TEXT,                    -- JSON 格式存储参数
    image_dir   TEXT,                    -- 图片目录路径
    crawl_time  DATETIME,
    status      TEXT DEFAULT 'pending'   -- pending/crawled/failed
);

-- 上架任务表
CREATE TABLE upload_tasks (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    product_id  INTEGER REFERENCES products(id),
    target      TEXT NOT NULL,           -- weixin/xiaohongshu/doudian
    upload_price REAL,                   -- 加价后价格
    status      TEXT DEFAULT 'pending',  -- pending/uploading/done/failed
    upload_time DATETIME,
    error_msg   TEXT
);
```

### 3.4 价格计算模块 (`pricing/`)

```
pricing/
├── __init__.py
└── calculator.py        # 价格计算
```

**支持的加价规则格式（后期可扩展）：**

| 规则格式 | 含义 | 示例 |
|----------|------|------|
| `+30%` | 加价百分比 | 100元 → 130元 |
| `+15` | 加固定金额 | 100元 → 115元 |
| `*1.5` | 倍率 | 100元 → 150元 |
| `=99.9` | 指定价格 | 直接用99.9元 |
| 空/未配置 | 使用默认规则 | 取全局配置 |

```python
def calculate_price(original: float, rule: str, default_rule: str = "+30%") -> float:
    rule = rule.strip() if rule else default_rule
    if rule.startswith("+") and rule.endswith("%"):
        return round(original * (1 + float(rule[1:-1]) / 100), 2)
    elif rule.startswith("+"):
        return round(original + float(rule[1:]), 2)
    elif rule.startswith("*"):
        return round(original * float(rule[1:]), 2)
    elif rule.startswith("="):
        return float(rule[1:])
    return original
```

**各平台上架流程概述：**

**微信视频号小店（已有基础）：**
1. 导航到商品发布页
2. 上传主图（Shadow DOM 内操作）
3. 填写标题 → 点击下一步
4. 填写面料/颜色/风格
5. 设置价格 → 提交

**小红书（新开发）：**
1. 登录小红书商家后台（`business.xiaohongshu.com`）
2. 进入商品管理 → 发布商品
3. 上传主图、填写标题
4. 选择类目、填写参数
5. 设置 SKU 和价格 → 提交审核

**抖店（新开发）：**
1. 登录抖店后台（`fxg.jinritemai.com`）
2. 进入商品管理 → 商品创建
3. 选择类目 → 填写基本信息
4. 上传图片、填写参数
5. 设置价格和库存 → 提交
